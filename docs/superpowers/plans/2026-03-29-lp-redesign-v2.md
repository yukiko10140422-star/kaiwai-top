# KAIWAI LP リデザイン v2 — 「サービスの価値が伝わるLP」

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** KAIWAIが「何のサービスか」「なぜ使うべきか」を3秒で伝えるLPに再構築する

**Architecture:** 現在のセクション構成を維持しつつ、コンテンツ（テキスト・構成・情報階層）とビジュアルの両方を刷新。LP構成のベストプラクティス（ファーストビュー→ボディ→クロージング）に基づく。

**Tech Stack:** Next.js 16, Tailwind CSS v4, TypeScript, inline style（Tailwind任意値バグ回避）

### 競合LPの共通パターン（dodaキャンパス・マイナビ・リクナビ）

| 要素 | 競合の実装 | KAIWAIの現状 | 改善点 |
|------|----------|------------|--------|
| ファーストビュー | 「弱みを強みに変える」等の具体ベネフィット見出し | 「迷っている学生へ」→ 曖昧 | **具体的なベネフィットを見出しに** |
| 結果プレビュー | 診断結果のビジュアル（円グラフ・カード）を事前表示 | なし | **結果イメージを見せて期待感を作る** |
| ステップ数明示 | 「30問」「10問」と明記 | 「5つの質問」と書いてあるが目立たない | **5問だけ！を最大の差別化に** |
| 信頼要素 | 企業ロゴ・利用者数・メディア掲載 | 「87%の学生が」のみ | **利用者数・大学名等を追加** |
| 感情共感 | 「自分らしさが見つからない」等の悩み訴求 | empathy chipsで一部実装 | **もっと目立つ位置・表現で共感** |
| CTA配置 | above the fold + セクション間 + 最後 | above the fold + 最後 | **中間CTAを追加** |

## セクション構成（改訂版）

```
1. Hero（ファーストビュー）
   → 3秒で「何のサービスか」「何が得られるか」が伝わる

2. Flow（サービスの仕組み）
   → 5問・3分・結果即表示の手軽さを強調

3. Result Preview（NEW: 結果イメージ）
   → 診断結果のモックアップを見せて「こういうのが出る」を可視化

4. Categories（4つのタイプ紹介）
   → 結果の種類を見せて興味を引く

5. GrowthPath（段階的支援の全体像）
   → 診断の先にある成長ストーリー

6. Features（特長）
   → 3分・無料・即結果・実務参加

7. Testimonials（体験談）
   → 実際に使った学生の声

8. FAQ（よくある質問）
   → 不安を解消

9. CTA（最終コンバージョン）
   → 背中を押す最後のひと声
```

## Task 1: Hero（ファーストビュー）の全面刷新

**Files:** `src/components/Hero.tsx`

### 現状の問題
- 見出し「迷っている学生へ。3分の診断で、次の一歩がわかる。」→ 何のサービスか不明確
- empathy chipsが小さく目立たない
- 写真がストック感
- 「5問だけ」という最大の差別化が埋もれている

### 改善方針（キャッチコピー8視点 × dodaキャンパス参照）

**Before:**
```
empathy chips: 何を始めればいいかわからない / ガクチカがない / 就活が不安
見出し: 迷っている学生へ。3分の診断で、次の一歩がわかる。
サブ: あなたの志向・興味・現在地から、4つのタイプに分類...
CTA: 無料で診断してみる
```

**After:**
```
empathy chips: → 削除。代わりにサブヘッド内で自然に触れる

見出し（問題解決 + 効果/成果の視点）:
「やりたいことが見つからない」を
たった5問で解決する。

サブヘッド（機能的価値 + 情緒的価値）:
5つの質問に答えるだけで、あなたの志向を4タイプに分類。
ガクチカ・留学・資格・サークル — それぞれに合った
「最初の一歩」を無料で提案します。

CTA: 無料で診断する（3分で完了）
CTAの下: 登録不要 ・ 完全無料 ・ 結果はすぐ表示
```

### 信頼バー（NEW）
Heroの下部に信頼要素を横一列で配置:
```
累計 1,200人 が診断済み ｜ 87%が行動を開始 ｜ 大学生満足度 4.6/5.0
```
（数値は仮。リリース前に実数値に差し替える想定）

### ビジュアル方針
- **写真グリッドを廃止** → 代わりに**診断結果のモックアップUI**をHeroビジュアルに
- スマホの画面に診断結果が表示されている風のイラスト/モックアップ
- これにより「何のサービスか」が一目でわかる

### 実装ステップ
- [ ] Step 1: 見出し・サブヘッド・CTAのテキスト差し替え
- [ ] Step 2: empathy chips → 削除
- [ ] Step 3: 写真グリッド → 診断結果モックアップUIに差し替え（CSSで作成）
- [ ] Step 4: 信頼バーを追加
- [ ] Step 5: ビルド確認 → コミット

  → KAIWAIの最大の特徴。診断後、実際のプロジェクトに参加して
