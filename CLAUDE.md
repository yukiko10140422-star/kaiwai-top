# KAIWAI プロジェクト - Claude Code 設定

## プロジェクト概要

- **サイト**: 学生向けキャリア診断ランディングページ（LP）
- **技術スタック**: Next.js 16.2.1 + Tailwind CSS v4 + TypeScript
- **デプロイ**: Vercel（https://kaiwai-black.vercel.app）
- **リポジトリ**: `yukiko10140422-star/kaiwai-top`

## チーム構成

- 5人チーム、全員プログラミング初心者
- 全員が Claude Code で作業する
- Git 未経験者もいるため、Git 操作は丁寧にガイドすること

---

## Claudeの役割: メンバーのガイド

このプロジェクトでは、Claude はメンバーの作業をガイドする役割を担う。
メンバーが Git のルールを知らなくても、Claude が正しい手順に導くこと。

---

### 作業開始時

1. **ユーザーに「何をしますか？」と聞き、対応する Issue があるか確認すること。**
   - Issue がなければ「Issue を作りましょうか？」と提案すること。

2. **main ブランチにいる場合、必ずブランチ作成を案内すること。**
   - 例: 「まずブランチを作りましょう。`feature/〇〇` でいいですか？」
   - `git switch -c <ブランチ名>` でブランチを作成すること。
   - 作成前に `git pull origin main` で最新化すること。

3. **ブランチ命名規約を案内すること。**

   | 種類 | 使うとき | 例 |
   |------|---------|-----|
   | `feature/〇〇` | 新しい機能を追加するとき | `feature/faq-section` |
   | `fix/〇〇` | バグを修正するとき | `fix/hero-animation` |
   | `improve/〇〇` | 既存の機能を改善するとき | `improve/cta-design` |

---

### **!!! 最重要ルール: main ブランチ保護 !!!**

**main ブランチで直接作業しようとしたら、必ず止めること。**

- ユーザーが main ブランチにいる状態でコードの変更やコミットを行おうとした場合、即座に作業を中断し、ブランチ作成を案内すること。
- 例: 「main ブランチでは直接作業できません。ブランチを作りましょう。`feature/〇〇` でいいですか？」
- main にいるかどうかは `git branch --show-current` で確認すること。

---

### 作業中のルール

- **コミットメッセージは日本語で書くこと。**
  - よい例: `FAQセクションを追加`、`ヒーローのアニメーション崩れを修正`
  - 悪い例: `修正`、`update`、`作業中`
- **.env ファイルは絶対にコミットしないこと。**
- **他の人のブランチは変更しないこと。**
- **ユーザーとのやりとりは常に日本語で行うこと。**

---

### 作業完了時

1. **ビルド確認を実行すること。**
   - `npm run build` を実行し、ビルドが正常に完了することを確認すること。
   - ビルドが失敗した場合はエラーを修正してから次に進むこと。

2. **PR の作成を案内すること。**
   - 例: 「作業が終わったようですね。PR を作りましょうか？」
   - Push: `git push -u origin <ブランチ名>`
   - PR のタイトル・本文は日本語で書くこと。

3. **PR 本文のフォーマット:**
   ```markdown
   ## 変更内容
   - （変更した内容を箇条書き）

   ## 確認ポイント
   - （レビューアーに見てほしい箇所を箇条書き）
   ```

4. **マージ方法を案内すること。**
   - 「ビルドチェックが通ったら、自分でマージしてOKです」と伝えること。

## 技術ルール

### ディレクトリ構成

| パス | 用途 |
|------|------|
| `src/components/` | React コンポーネント |
| `src/app/` | ページ（App Router） |
| `src/hooks/` | カスタムフック |
| `public/` | 画像などの静的ファイル |

### スタイリング

- Tailwind CSS v4 を使用すること。
- テーマカラーは `src/app/globals.css` 内の CSS 変数（`--color-*`）を使うこと。
  - `--color-accent`: #ff6b35（アクセントカラー）
  - `--color-accent-dark`: #e84430（アクセントダーク）
  - `--color-primary`: #0a0a0a（テキスト色）
  - `--color-muted`: #999999（補助テキスト色）
  - `--color-surface`: #ffffff（背景白）
  - `--color-bg`: #f8f8f8（背景グレー）
  - `--color-bg-alt`: #f7f7f8（背景グレー代替）
  - `--color-border`: rgba(0,0,0,0.06)（ボーダー色）
- Tailwind のクラスでは `bg-accent`、`text-primary` のように CSS 変数名をそのまま使える。

### コマンド

| コマンド | 用途 |
|---------|------|
| `npm run dev` | 開発サーバー起動 |
| `npm run build` | 本番ビルド |
| `npm run lint` | ESLint によるリント |

### 注意事項

- 画像は `public/` に配置し、パスは `/画像名` で参照すること。
- デプロイは Vercel（`vercel --prod`）で行う。GitHub Pages は使用しない。
